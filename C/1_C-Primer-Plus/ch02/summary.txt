프로그래밍은 컴퓨터에게 특정한 명령들을 내려주기 위한 일련의 활동이다.
C 프로그래밍은 C 언어 표준에 맞춰서 명령들을 작성하는 것이라고 할 수 있다.

C 언어의 기본 모듈은 함수이다.
C 언어는 함수 단위로 명령을 작성한다.
C 프로그램은 반드시 main()이라는 함수를 필요로 한다. main()은 프로그램이 처음 시작할 때 호출되는 함수이다.

C 언어는 컴파일 언어이다.
C 프로그램은 C 컴파일러에 의해 컴퓨터가 이해할 수 있는 기계어로 변환된다. 변환된 코드를 실행코드라 한다.

프로그래밍 언어 표준에 맞춰서 작성한 코드를 소스코드라 한다.
컴파일러는 컴파일 과정에서 작성된 소스코드의 문법 오류(syntax error)를 탐지할 수 있다.
그러나, 의미 오류(semantic error)는 탐지할 수 없으며 그 것을 확인하기 위해서는 프로그램 상태를 추적하는 등의 노력이 필요하다.

C 환경에서 실행코드가 생성되는 과정을 정리하면 아래와 같다.
-. 소스코드 => 오브젝트코드 (w/ 컴파일러)
-. 오브젝트코드 + 라이브러리코드 + 시동코드 => 실행코드 (w/ 링커)
